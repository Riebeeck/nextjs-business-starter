## Technical Implementation

### App Router Architecture

The project leverages Next.js 15's App Router for a modern, file-based routing system:

```typescript
app/
├── layout.tsx          # Root layout with theme provider
├── page.tsx            # Home page
├── about/
│   └── page.tsx        # About page
├── blog/
│   ├── page.tsx        # Blog listing
│   └── [slug]/
│       └── page.tsx    # Dynamic blog posts
└── projects/
    ├── page.tsx        # Projects listing
    └── [slug]/
        └── page.tsx    # Dynamic project pages
```

### Content Management

Blog posts and projects are managed through MDX files with frontmatter:

```typescript
// lib/content.ts
export async function getAllBlogPosts(): Promise<BlogPost[]> {
  const blogDir = path.join(process.cwd(), 'content/blog');
  const fileNames = fs.readdirSync(blogDir);
  
  const posts = fileNames
    .filter((fileName) => fileName.endsWith('.mdx'))
    .map((fileName) => {
      const fullPath = path.join(blogDir, fileName);
      const fileContents = fs.readFileSync(fullPath, 'utf8');
      const { data, content } = matter(fileContents);
      
      return {
        slug: fileName.replace(/\.mdx$/, ''),
        ...data,
        content,
      } as BlogPost;
    });
    
  return posts.sort((a, b) => 
    new Date(b.date).getTime() - new Date(a.date).getTime()
  );
}
```

### Dark Mode Implementation

Using next-themes for a seamless dark mode experience:

```typescript
// components/ThemeProvider.tsx
'use client';

import { ThemeProvider as NextThemesProvider } from 'next-themes';

export default function ThemeProvider({ children }) {
  return (
    <NextThemesProvider attribute="class" defaultTheme="system" enableSystem>
      {children}
    </NextThemesProvider>
  );
}
```

## Challenges and Solutions

### Challenge 1: MDX Rendering

**Problem:** Rendering MDX content with custom components and syntax highlighting.

**Solution:** Created a custom MDX components map with styled elements and integrated rehype-pretty-code for beautiful code blocks.

### Challenge 2: Theme Persistence

**Problem:** Preventing flash of unstyled content on page load with dark mode.

**Solution:** Used next-themes with `suppressHydrationWarning` and proper class-based dark mode strategy.

### Challenge 3: Performance Optimization

**Problem:** Keeping bundle size small while maintaining rich functionality.

**Solution:** Leveraged Server Components for static content, only using Client Components where interactivity is needed.

2. **TypeScript prevents bugs** - Strict typing caught numerous potential issues during development
3. **Content-first approach** - MDX files make content management simple and version-controlled
4. **Dark mode is expected** - Users appreciate the option and it's becoming a standard feature
